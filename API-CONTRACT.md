# API CONTRACT

## Auth
### `POST /auth/signup`
Creates a user and immediately returns a Bearer token.

### `POST /auth/login`
Returns a Bearer token unless the user is disabled or suspended, in which case the response is `403`.

### `GET /me`
Returns the authenticated profile, avatar/thumb metadata, ghost mode, admin flag, trial expiry, and best leaderboard badge.

## Presence
### Timestamp and cursor unit
- Presence delta cursors use **milliseconds**.
- The documented incremental query parameter is `updated_since_ms`.

### `POST /presence/update`
Body:
```json
{
  "lat": 40.75,
  "lng": -73.99,
  "heading": 90,
  "accuracy": 5
}
```
Response shape:
```json
{
  "ok": true,
  "cursor": 1710000000000
}
```

### `GET /presence/all`
Compatibility snapshot route. Supports bounds and returns:
- `ok`
- `count`
- `items`
- `online_count`
- `ghosted_count`
- `visible_count`

This route stays available as a backward-compatible fallback, but `/presence/viewport` and `/presence/delta` are the preferred optimized contracts.

### `GET /presence/viewport`
Two modes:
- snapshot when `updated_since_ms` is omitted or `0`
- delta when `updated_since_ms` is provided

Snapshot responses include:
- `ok`
- `mode = "snapshot"`
- `count`
- `items`
- `removed` (always `[]` for snapshots)
- `cursor`
- `server_time_ms`
- `online_count`
- `ghosted_count`
- `visible_count`
- `viewport`

When `updated_since_ms` is supplied, `/presence/viewport` delegates to the same optimized delta contract as `/presence/delta`.

### `GET /presence/delta`
Parameters:
- `updated_since_ms`
- `max_age_sec`
- `min_lat`
- `min_lng`
- `max_lat`
- `max_lng`
- `zoom`
- `include_removed`
- `limit`

Stable response fields:
- `ok`
- `mode = "delta"`
- `items`
- `removed`
- `cursor`
- `server_time_ms`
- `online_count`
- `ghosted_count`
- `visible_count`

Returns only changed visible rows plus tombstones/removals when requested. The incremental query parameter remains `updated_since_ms`, and `removed` entries are how ghosting, deletes, suspension, or stale users are surfaced to optimized clients.
Removal reasons currently include values such as:
- `ghost_mode`
- `disabled`
- `suspended`
- `stale`
- `account_deleted`

### `GET /presence/summary`
Returns aggregate counts only.

## Safe Phase 2 live chat
### `GET /chat/live/capabilities`
Requires normal Bearer auth and returns compact browser-safe SSE connection info:
```json
{
  "ok": true,
  "sse_enabled": true,
  "public": { "enabled": true, "url": "https://.../chat/public/events?live_token=...", "stream": "public" },
  "private": { "enabled": true, "url": "https://.../chat/private/events?live_token=...", "stream": "private" },
  "ttl_seconds": 60,
  "heartbeat_seconds": 15,
  "live_token_ttl_seconds": 60,
  "recovery": {
    "last_event_id_header": "Last-Event-ID",
    "public_summary_url": "/chat/public/summary",
    "private_summary_url": "/chat/private/summary",
    "dm_thread_summary_url_template": "/chat/dm/{other_user_id}/summary"
  }
}
```

`ttl_seconds` is the main frontend-facing ticket lifetime field. `live_token_ttl_seconds` is retained as a compatibility alias.

### SSE auth rules
The SSE endpoints accept either:
- a normal `Authorization: Bearer ...` header, or
- a short-lived signed `live_token` query param generated by `/chat/live/capabilities`.

The signed ticket is scoped to the authenticated user id plus a stream scope (`public` or `private`) and expires after a short 30-90 second window by default. The long-lived login token is **not** intended to be placed directly into EventSource query strings.

### `GET /chat/public/events`
Public/global live stream emitting compact `chat.message` events.

### `GET /chat/private/events`
Per-user DM summary stream emitting compact `dm.thread_updated` and `dm.unread_changed` events.

### `GET /chat/rooms/{room}/events`
Room-specific additive stream; available but not required for the main frontend contract.
